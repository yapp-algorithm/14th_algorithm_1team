#DFS(Depth First Search)
* 깊이 우선 탐색 방법
* 이름 그대로 한쪽 노드를 타고 가장 깊은 곳까지 갔다가, 더이상 연결된 노드가 없으면 다시 돌아 나온 후<br>
또 다른 엣지를 타고 가장 깊은 노드까지 가는 과정을 반복하며 그래프를 탐색

> 단점
>- 특정 문제의 경우 노드가 무한히 깊어져서 빠져나오지 못하게 될 수 있음
>- 최단 거리를 구하는 문제의 경우 처음으로 목적지를 만나도 그 경로가 최단거리인지 판단 X

* 실행과정
> 재귀
>- 노드를 방문하고 재귀적으로 모든 인접한 노드를 방문(깊이 우선으로)
>- 만약 인접한 노드가 이미 방문했던 것이라면, 백트래킹(재귀함수를 빠져나감) 함으로써 무한루프를 방지
>- 만약 모든 인접한 노드를 방문했으면 그 노드는 탐색 종료
>- Python의 경우 재귀의 최대 깊이가 정해져있기 때문에 특정 경우 에러가 남

> stack 이용
>- 현재 노드와 인접한 노드 중 방문하지 않은 모든 노드들을 스택에 넣음
>- 현재 노드에 방문 표시를 한 후, 스택에서 가장 바깥의 원소를  빼서 첫 번째 과정을 반복
>- 스택이 비면 탐색 종료


#BFS
* 너비 우선 탐색 방법
* DFS가 stack을 이용했다면 BFS는 queue를 이용
* 큐에서 현재 노드와 연결되어 있는 노드들을 차례차례 넣고, 큐에서 순차적으로 노드들을 빼면서<br>
하나하나 탐색
* DFS와 비교했을 때 가장 큰 장점은 가중치가 없는 그래프에서 목적지를 찾게 되면 그때의 탐색 레벨이<br>
목적지까지의 최단 거리가 되어서 의미없는 탐색을 줄일 수 있음
* 최악의 경우에는 DFS나 BFS나 걸리는 시간은 같음

* BFS를 실행할 때, 출발점으로부터 거치는 edge의 수를 조금 더 쉽게 알기 위해서 layer라는 것을 정의(선택사항)
> Layer의 귀납적 정의
>- L[0] := {s}
>- L[1] := L[0]와 이웃한 모든 노드들의 집합
>- L[i+1] := L[0], L[1], ..., L[i]에 포함되지 않으면서 L[i]와 이웃한 모든 노드들의 집합
